## 关于程序不输出（刷新输出）

# 关于程序不输出（刷新输出）


在C语言中，输出通常会被缓冲，这意味着printf函数的输出可能不会立即显示在屏幕上，而是被存储在内部缓冲区中，直到缓冲区被刷新。缓冲区刷新通常发生在以下几种情况下：

1. **缓冲区满时**：当输出缓冲区达到一定容量时，它会自动刷新。
2. **遇到换行符时**：在许多环境中，printf函数在输出换行符（`\n`）时会刷新缓冲区。
3. **程序正常结束时**：当程序执行完毕并正常退出时，所有的输出缓冲区都会被刷新。
4. **显式刷新**：可以使用fflush函数显式地刷新输出缓冲区，例如`fflush(stdout);`。
5. **输入操作时**：在某些实现中，进行输入操作（如调用scanf函数）可能会刷新输出缓冲区。
6. **遇到特定函数调用时**：某些函数调用，如perror，可能会刷新标准错误输出缓冲区。在您的代码中，由于没有包含换行符，并且没有显式刷新缓冲区，如果程序在输出之前异常终止（例如由于无限循环或错误），那么缓冲区中的内容可能不会被刷新到屏幕上，导致看起来好像没有输出。为了确保输出即使在没有换行符的情况下也能被刷新，您可以在printf调用后使用fflush函数，如下所示：`cprintf("%d", i);fflush(stdout); // 显式刷新标准输出缓冲区`这样，即使程序没有正常结束，输出也会被立即显示在屏幕上。

## switch case语句

# switch case语句


## 特性：


Q：为什么case后无break语句会执行其他case中的语句


问题代码：


```c
switch(i)
{
  case a:
  printf("文本一\n")；
  case b：
  printf("文本二\n")；
}
```


执行结果：（_假设前提条件符合第一个case语句执行条件）_


```plain text
文本一
文本二
```


A: 这是C语言中 switch 语句的设计特性，主要是为了允许多个 case 共享相同的代码块。（无break语句会导致编译器识别为如下形式）↓


```c
switch(i)
{
  case a:
  case b:
    printf("文本一\n")；
    printf("文本二\n")；
}
```
